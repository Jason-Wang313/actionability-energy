\section{Low-Compute Benchmark and Experiments}

\paragraph{Environments.}
The benchmark uses four deterministic CPU environments. The point mass is a positive control with $J=I$. The nonholonomic car exposes sideways visual futures that are locally impossible. The two-link arm uses an end-effector Jacobian and action limits. The planar pusher has a contact-dependent Jacobian, so object motion before contact is non-actionable.

\paragraph{Generated futures.}
We use controlled future generators rather than large video models: straight-line interpolation, noisy splines, passive smoothness priors, scripted rollouts, and intentionally non-actionable futures. These generators are proxies for generated visual futures and are not claimed to match real video-model distributions.

\paragraph{Experiment 1: constructive counterexample.}
The car example in Proposition 1 is implemented directly. A sideways future has the same endpoint smoothness as a forward future under passive scoring, but the residual rejects it because the displacement lies outside the car's local tangent set.

\paragraph{Experiment 2: failure prediction with hostile baselines.}
For each candidate, we compute residuals, recover controls, roll out the true environment, and mark failure when the executed path diverges from the imagined path. We compare analytic actionability, learned-J actionability, noisy learned-J actionability, wrong-map residual, inverse-dynamics reconstruction error, a learned classifier, a forward-inverse asymmetry proxy, smoothness, path length, action norm, passive likelihood proxy, and random score.

\paragraph{Experiment 3: repair.}
We compare the original candidate and goal-only, world+goal, smoothness-regularized, actionability-regularized, full-energy, learned-J, noisy learned-J, and wrong-map repair variants. Adding actionability changes the trajectory rather than only rescoring it; it encourages futures whose local increments can be produced by bounded actions.

\paragraph{Experiment 4: embodiment swap.}
We score the same local $(dx,dy)$ displacement under different action maps. The residual changes sharply across embodiments, showing that executability is body-conditioned.

\begin{figure}[t]
\centering
\includegraphics[width=0.95\linewidth]{figures/fig6_embodiment_swap.pdf}
\caption{Embodiment swap residual fields. The same representation displacement can be actionable for one body and non-actionable for another. Low residual regions now show the feasible tangent structure without collapsing into unreadable black regions.}
\label{fig:swap}
\end{figure}

\paragraph{Experiment 5: visual/keypoint futures.}
To avoid a purely state-space story, we render CPU keypoint representations for the car and pusher and compute residuals in keypoint space using finite-difference or learned visual Jacobians. This remains a controlled proxy rather than a large video model, but it tests the intended representation-level interface.
